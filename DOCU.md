#### Setting up Wordpress Project on Local Environment
1. Download wordpress setup [WORDPRESS](http://breakdance.io)
2. Place the "wordpress_setup" folder in "/Desktop/Projects/wordpress/"
3. Rename the "wordpress_setup" folder with the project name
4. Setup up virtual host - [Local Domain Setup](https://git.ceb02.ds.network/development-hub/technology-stack#development-environment)
5. Create a database on Local
    1. Access [Local phpmyadmin](http://localhost/phpMyAdmin/)
    2. Create Database and name it after your project name and add prefix "sam_projectname" on it.
    3. Create a database user named "sam_dev" password "S/852*963." 
    4. Select All Global Priveleges for the user.
    5. Click "Go" button.
6. GitKraken Setup
   1. Install [GitKraken](https://support.gitkraken.com/how-to-install/)
   2. Go to GitKraken -> Preferences -> Authentication -> General -> Click Generate new Private/Public Key
7. Add the SSH Key generated by GitKraken to GitLab account
    1. Locate your SSH keys in "/Users/employee.name/Documents/"
    2. Copy the SSH public key "gitkraken_rsa.pub"
    3. Add the SSH public key on [GitLab](https://git.ceb02.ds.network/profile/keys) and put title (NameMacMini)
8. Create a project repository on Gitlab **under the group it should be assigned to**.
    1. Create new project
    2. Add Project Name
9. Clone Repository in Git Kraken

#### Setting up Wordpress Project on Staging Environment
1. Create a staging website for your project.
    1.  Access [CPANEL (No. 1)](https://git.ceb02.ds.network/divi-theme/dev-cycle#uploading-on-live)
    2.  In Domains Section -> Click Subdomains
    3.  Create a subdomain by inputting the project name in **subdomain field**
    4.  Click "Create" button
2. Create database for your staging website.
    1. In Databases Section -> Click MySQL Databases
    2. Create New Database -> input the project name in **New Database field**
    3. Click "Create Database"
3. Adding privelege to a database user for your database.
    1. In Add User to Database.
    2. Select "sam_dev" as **USER**
    3. Select the database you created and click add.
    4. In **Manage User Privileges** -> Check **ALL PRIVILEGES** -> Click "Make Changes".
4. Follow instructions [here (No.3)](https://git.ceb02.ds.network/divi-theme/dev-cycle#uploading-on-live).
